## Technical Context

### Architecture Reference

The current `MasterTemplateEditor` flattens all groups with `protocols(in:)` — `section.groups.flatMap(\.protocols).sorted { $0.position < $1.position }`. This story replaces that with two nested `ForEach` loops: one over groups (stacks), one over protocols within each group.

### Files to Modify

| File | Purpose | Changes |

## Implementation Guidance

### Approach

Replace the flat `ForEach(protos)` inside each section with a two-level loop: `ForEach(section.sortedGroups)` → `ForEach(group.sortedProtocols)`. Add `HabitStackHeader` between section header and protocol rows (conditionally hidden for single-stack sections). Keep `onDelete` on the inner `ForEach`.

### Layer Sequence

1. UI layer only — modify `MasterTemplateEditor.swift`

### Step-by-Step

1. **Remove the `protocols(in:)` helper** (line 14-16):
```swift
// DELETE this:
private func protocols(in section: ProtocolSection) -> [UserProtocol] {
    section.groups.flatMap(\.protocols).sorted { $0.position < $1.position }
}
```

2. **Add collapsing helper:**
```swift
private func shouldCollapseStack(_ section: ProtocolSection, _ group: ProtocolGroup) -> Bool {
    section.groups.count == 1 && group.name == section.name
}
```

3. **Replace the section body** (lines 28-57). Current:
```swift
ForEach(sections) { section in
    Section {
        let protos = protocols(in: section)
        if protos.isEmpty { /* empty text */ }
        ForEach(protos) { proto in
            NavigationLink { ... } label: { ProtocolRow(proto: proto) }
        }
        .onDelete { ... }
    } header: {
        sectionHeader(section)
    }
}
```

Replace with:
```swift
ForEach(sections) { section in
    Section {
        ForEach(section.sortedGroups) { group in
            // Stack header (hidden for single-stack with matching name)
            if !shouldCollapseStack(section, group) {
                HabitStackHeader(
                    name: group.name,
                    completedCount: 0,  // My Protocols doesn't track completion
                    totalCount: group.protocols.count
                )
            }

            let protos = group.sortedProtocols
            if protos.isEmpty {
                EmptyStackPlaceholder()
            }

            ForEach(protos) { proto in
                NavigationLink {
                    ProtocolDetailView(
                        protocolId: proto.id.uuidString,
                        label: proto.label,
                        subtitle: proto.subtitle,
                        documentId: proto.documentId
                    )
                } label: {
                    ProtocolRow(proto: proto)
                }
            }
            .onDelete { offsets in
                for offset in offsets { modelContext.delete(protos[offset]) }
            }
        }
    } header: {
        sectionHeader(section)
    }
}
```

4. **Update `ProtocolRow` navigation** — the existing `ProtocolRow` struct (line 173) stays unchanged. It already shows icon, label, subtitle, duration.

5. **Verify `sortedGroups` and `sortedProtocols`** exist on the models. From the model exploration, `ProtocolSection` has a `sortedGroups` computed property and `ProtocolGroup` has a `sortedProtocols` computed property. Confirm they sort by `position`.

6. **Stack header on My Protocols:** The `completedCount` for the My Protocols page is always 0 (it's the template, not the daily snapshot). The header here shows the total protocol count, not completion. Adjust the header to accept an optional mode or just pass `totalCount: group.protocols.count` and `completedCount: 0`. This means the header will show "0/N" — **or better**: on My Protocols, the header should show just the count without a fraction. Add a `showCompletion: Bool` parameter to `HabitStackHeader`:

```swift
struct HabitStackHeader: View {
    let name: String
    let completedCount: Int
    let totalCount: Int
    var showCompletion: Bool = true  // false on My Protocols

    var body: some View {
        HStack {
            Text(name.uppercased()) ...
            if showCompletion {
                // ... completion badge
            } else if totalCount > 0 {
                Text("\(totalCount)")
                    .font(.caption2.monospacedDigit())
                    .foregroundStyle(.tertiary)
            }
            Spacer()
        }
    }
}
```

On My Protocols: `HabitStackHeader(name: ..., completedCount: 0, totalCount: N, showCompletion: false)`
On Daily Today: `HabitStackHeader(name: ..., completedCount: X, totalCount: N)` (default showCompletion: true)

### Code Patterns to Follow

```swift
// Existing section header pattern (MasterTemplateEditor, line 112):
private func sectionHeader(_ section: ProtocolSection) -> some View {
    HStack {
        Text(section.name)
            .font(.subheadline.weight(.semibold))
        Spacer()
        Menu { ... }
    }
}

// Stack header goes INSIDE the Section body, before protocol rows.
// It's visually subordinate to the section header.
```

### Gotchas / Watch Out For

- **`ForEach` inside `Section` inside `List`:** SwiftUI `List` handles this nesting well. Each `ForEach` gets its own identity scope. Use model `.id` (UUID) for stable identity.
- **`onDelete` scope:** The `onDelete` must be on the inner `ForEach(protos)`, not on `ForEach(section.sortedGroups)`. Deleting from the group-level ForEach would delete groups, not protocols.
- **`sortedGroups` returns a new array:** It's a computed property, not a stored array. Each call creates a new sorted copy. This is fine for small N but avoid calling it in tight loops.
