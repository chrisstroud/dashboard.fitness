## Technical Context

### Architecture Reference

Position sync is optimistic: local SwiftData write happens immediately (Story 7.1), then the API PATCH fires asynchronously. If the PATCH fails, the next full `syncProtocols()` pull will overwrite local positions with server values — but this is acceptable since the user will likely have made the same reorder again.

## Implementation Guidance

### Approach

Add a `syncPositions()` method to `SyncService` that accepts optional arrays of sections, groups, and protocols, serializes their IDs and positions, and PATCHes to the server. Then wire it into the three reorder handlers in `MasterTemplateEditor`.

### Layer Sequence

1. Service layer — add method to `SyncService.swift`
2. UI layer — call from reorder handlers in `MasterTemplateEditor.swift`

### Step-by-Step

1. **Add `syncPositions()` to SyncService:**

```swift
// MARK: - Position Sync

func syncPositions(
    sections: [ProtocolSection]? = nil,
    groups: [ProtocolGroup]? = nil,
    protocols: [UserProtocol]? = nil
) {
    guard let token = AuthService.shared.token else { return }

    var body: [String: Any] = [:]
    if let sections {
        body["sections"] = sections.map {
            ["id": $0.id.uuidString, "position": $0.position]
        }
    }
    if let groups {
        body["groups"] = groups.map {
            var dict: [String: Any] = ["id": $0.id.uuidString, "position": $0.position]
            if let sectionId = $0.section?.id.uuidString {
                dict["section_id"] = sectionId
            }
            return dict
        }
    }
    if let protocols {
        body["protocols"] = protocols.map {
            var dict: [String: Any] = ["id": $0.id.uuidString, "position": $0.position]
            if let groupId = $0.group?.id.uuidString {
                dict["group_id"] = groupId
            }
            return dict
        }
    }

    guard !body.isEmpty else { return }

    Task {
        #if DEBUG
        let baseURL = "http://localhost:5001"
        #else
        let baseURL = "https://dashboardfitness-production.up.railway.app"
        #endif

        var request = URLRequest(url: URL(string: "\(baseURL)/api/protocols/reorder")!)
        request.httpMethod = "PATCH"
        request.setValue("Bearer \(token)", forHTTPHeaderField: "Authorization")
        request.setValue("application/json", forHTTPHeaderField: "Content-Type")
        request.httpBody = try? JSONSerialization.data(withJSONObject: body)
        _ = try? await URLSession.shared.data(for: request)
    }
}
```

2. **Wire into MasterTemplateEditor reorder handlers** (from Story 7.1):

```swift
private func moveSectionUp(_ section: ProtocolSection) {
    let sorted = sections.sorted { $0.position < $1.position }
    guard let idx = sorted.firstIndex(where: { $0.id == section.id }), idx > 0 else { return }
    sorted[idx].position = idx - 1
    sorted[idx - 1].position = idx
    SyncService.shared.syncPositions(sections: [sorted[idx], sorted[idx - 1]])
}

private func moveStacks(_ indices: IndexSet, _ destination: Int, in section: ProtocolSection) {
    var ordered = section.sortedGroups
    ordered.move(fromOffsets: indices, toOffset: destination)
    for (i, group) in ordered.enumerated() { group.position = i }
    SyncService.shared.syncPositions(groups: ordered)
}

private func moveProtocols(_ indices: IndexSet, _ destination: Int, in group: ProtocolGroup) {
    var ordered = group.sortedProtocols
    ordered.move(fromOffsets: indices, toOffset: destination)
    for (i, proto) in ordered.enumerated() { proto.position = i }
    SyncService.shared.syncPositions(protocols: ordered)
}
```

### Gotchas / Watch Out For

- **`baseURL` duplication:** Multiple files hardcode the base URL. Follow the existing pattern — don't refactor to a shared constant in this story.
- **Race condition:** User reorders → PATCH fires → full sync triggers → server has old positions → full sync overwrites local. Mitigation: this is acceptable for a personal app. The user will notice and re-drag. A future improvement could skip position overwrites during `syncProtocols` if a PATCH is in-flight.
- **Serialization:** `JSONSerialization.data(withJSONObject:)` requires `[String: Any]` dictionaries. The `map` closures produce this correctly.

## Testing Notes

### Test Scenarios
1. Reorder protocols → check server DB shows updated positions (via API GET or direct DB query)
2. Reorder stacks → server positions match
3. Move section → server positions match
4. Kill app after reorder, relaunch → positions persist (SwiftData)
5. Trigger full sync after reorder → positions should match (PATCH + sync both go to same server)

### Edge Cases
- No network: PATCH fails silently, local positions correct, next sync overwrites
- Rapid reorders: multiple PATCHes in flight — last one wins on server, all are correct
