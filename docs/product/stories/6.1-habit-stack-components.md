## Technical Context

### Architecture Reference

Both My Protocols and Daily Today need to render stack cards. The data sources differ — `ProtocolGroup` (SwiftData model) on My Protocols, `DailyGroup` (struct) on Daily Today. A shared view interface allows both pages to use the same card component.

## Implementation Guidance

### Approach

Create a single file with `HabitStackHeader` and two card variants — one for My Protocols (SwiftData, editable) and one for Daily Today (struct-based, check-off only). Use a simple protocol to share the completion-counting logic. Keep it under 150 lines.

### Layer Sequence

1. UI layer only — new SwiftUI view file

### Step-by-Step

1. Create `HabitStackCard.swift` with the following structure:

```swift
import SwiftUI
import SwiftData

// MARK: - Habit Stack Header

struct HabitStackHeader: View {
    let name: String
    let completedCount: Int
    let totalCount: Int

    private var allCompleted: Bool { totalCount > 0 && completedCount == totalCount }

    var body: some View {
        HStack(alignment: .center) {
            Text(name.uppercased())
                .font(.caption2.bold())
                .foregroundStyle(.secondary)
                .tracking(0.6)

            if totalCount == 0 {
                // empty — no badge
            } else if allCompleted {
                Image(systemName: "checkmark.circle.fill")
                    .font(.caption2)
                    .foregroundStyle(.green)
                Text("All done")
                    .font(.caption2)
                    .foregroundStyle(.green)
            } else {
                Text("\(completedCount)/\(totalCount)")
                    .font(.caption2.monospacedDigit())
                    .foregroundStyle(.tertiary)
            }

            Spacer()
        }
        .padding(.horizontal, 4)
        .padding(.top, 8)
        .padding(.bottom, 2)
    }
}
```

2. For the **empty stack** placeholder:

```swift
struct EmptyStackPlaceholder: View {
    var body: some View {
        HStack {
            Spacer()
            Text("No protocols — tap + to add one")
                .font(.caption)
                .foregroundStyle(.tertiary)
            Spacer()
        }
        .padding(.vertical, 12)
    }
}
```

3. **Key design decision:** The card container (rounded rect + material) is applied by the *parent view* (MasterTemplateEditor or DailySectionView), not by HabitStackCard itself. This is because the parent controls padding, onMove, and onDelete modifiers which must wrap the ForEach content. The shared components are the **header** and **empty placeholder** — lightweight building blocks, not full cards.

4. Add SwiftUI previews:

```swift
#Preview("Stack with protocols") {
    VStack {
        HabitStackHeader(name: "Wake Up", completedCount: 2, totalCount: 4)
        // ... mock rows
    }
    .background(.regularMaterial, in: RoundedRectangle(cornerRadius: 12, style: .continuous))
    .padding()
}

#Preview("Completed stack") {
    HabitStackHeader(name: "Mindset", completedCount: 3, totalCount: 3)
        .padding()
}

#Preview("Empty stack") {
    VStack {
        HabitStackHeader(name: "New Stack", completedCount: 0, totalCount: 0)
        EmptyStackPlaceholder()
    }
    .padding()
}
```

### Code Patterns to Follow

```swift
// Existing section header pattern from DailySectionView (HomeTab.swift, line 182):
HStack(alignment: .center) {
    Text(section.name.uppercased())
        .font(.caption.bold())
        .foregroundStyle(.secondary)
        .tracking(0.8)
    // ... completion badges
    Spacer()
}
.padding(.horizontal, 20)

// Stack header is similar but:
// 1. Slightly smaller font (.caption2 vs .caption)
// 2. Less horizontal padding (nested inside card)
// 3. Same completion badge pattern
```

### Gotchas / Watch Out For

- **Don't over-abstract.** The temptation is to create a generic `StackCardView<Content>` with `@ViewBuilder`. Resist — the two pages have different content (NavigationLink rows vs check-off rows). Share the header and placeholder; let each page compose its own card.
- **Padding coordination:** The card's `.padding(.horizontal, 16)` is applied by the parent (matching existing `DailySectionView` pattern). The header's internal padding is smaller since it's inside the card.

## Dev Notes

### Architecture Constraints
- Components must work with both `ProtocolGroup` (SwiftData) and `DailyGroup` (struct) data
- Keep components stateless — no `@Environment(\.modelContext)`, no mutation logic
- Both pages will apply their own card container (material + rounded rect)

### Patterns to Follow
- Follow existing `DailySectionView` styling for visual consistency
- Use `.caption2` for stack-level text (one step smaller than section `.caption`)
- Use `.tracking(0.6)` for uppercased stack names (slightly tighter than section `.tracking(0.8)`)
