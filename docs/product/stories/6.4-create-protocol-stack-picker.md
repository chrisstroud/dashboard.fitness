## User Story

**As a** dashboard user
**I want to** choose which habit stack a new protocol belongs to
**So that** I can place protocols in the right behavioral chain without editing afterward

---

## Acceptance Criteria

- [ ] **AC1:** Stack picker appears only when the selected section has 2+ groups
- [ ] **AC2:** When a section has exactly 1 group, the protocol is auto-assigned to that group (picker hidden)
- [ ] **AC3:** `initialSection` pre-selection still works (from section "…" menu)
- [ ] **AC4:** New `initialStack` parameter allows pre-selecting a stack (from stack "…" menu in Epic 8)
- [ ] **AC5:** Save flow uses the selected group (stack) instead of `section.groups.first`
- [ ] **AC6:** The "Add to" section in the form shows section picker → optional stack picker in natural reading order

## Technical Context

### Architecture Reference

The current `CreateProtocolSheet` has a section picker and finds/creates a default group in the save flow. This story adds a conditional stack picker between the section picker and the type picker.

## Implementation Guidance

### Approach

Add a `@State private var selectedGroupId: UUID?` alongside the existing `selectedSectionId`. When the selected section has 2+ groups, show a stack picker. When it has 1 group, auto-select it. Update the save flow to use the selected group directly.

### Layer Sequence

1. UI layer only — modify `CreateProtocolSheet.swift`

### Step-by-Step

1. **Add new state and parameters:**

```swift
struct CreateProtocolSheet: View {
    var initialSection: ProtocolSection?
    var initialStack: ProtocolGroup?          // ← NEW
    let onSave: () -> Void

    @State private var selectedSectionId: UUID?
    @State private var selectedGroupId: UUID?  // ← NEW
    // ... existing state
```

2. **Add computed properties:**

```swift
private var selectedSection: ProtocolSection? {
    sections.first { $0.id == selectedSectionId }
}

private var selectedGroup: ProtocolGroup? {        // ← NEW
    selectedSection?.groups.first { $0.id == selectedGroupId }
}

private var sectionHasMultipleStacks: Bool {       // ← NEW
    (selectedSection?.groups.count ?? 0) > 1
}
```

3. **Add stack picker in the form** (after section picker, before type picker):

```swift
// Section picker (existing, unchanged)
if initialSection == nil {
    Section("Add to") {
        Picker("Section", selection: $selectedSectionId) {
            ForEach(sections) { section in
                Text(section.name).tag(section.id as UUID?)
            }
        }
        .pickerStyle(.menu)

        // Stack picker — only when section has multiple stacks
        if sectionHasMultipleStacks {
            Picker("Habit Stack", selection: $selectedGroupId) {
                ForEach(selectedSection?.sortedGroups ?? []) { group in
                    Text(group.name).tag(group.id as UUID?)
                }
            }
            .pickerStyle(.menu)
        }
    }
}
```

4. **Auto-select group when section changes:**

```swift
.onChange(of: selectedSectionId) { _, newValue in
    // Auto-select first group when section changes
    if let section = sections.first(where: { $0.id == newValue }) {
        selectedGroupId = section.sortedGroups.first?.id
    }
}
```

5. **Update `onAppear`:**

```swift
.onAppear {
    selectedSectionId = initialSection?.id ?? sections.first?.id
    selectedGroupId = initialStack?.id ?? selectedSection?.sortedGroups.first?.id
}
```

6. **Update `save()` method** — replace the group find/create logic (lines 106-114) with:

```swift
private func save() {
    guard let group = selectedGroup ?? selectedSection?.groups.first else { return }
    isSaving = true
    // ... rest of save flow uses `group` directly
```

7. **Update Save button disabled state:**

```swift
Button("Save") { save() }
    .disabled(label.isEmpty || selectedSection == nil || isSaving)
    .bold()
```

The `selectedGroup` check isn't needed in `disabled` because `selectedSection != nil` guarantees at least one group exists (sections always have a default group).

### Code Patterns to Follow

```swift
// Existing section picker pattern (CreateProtocolSheet, line 31-38):
Picker("Section", selection: $selectedSectionId) {
    ForEach(sections) { section in
        Text(section.name).tag(section.id as UUID?)
    }
}
.pickerStyle(.menu)

// Stack picker follows identical pattern, just one level deeper
```

### Gotchas / Watch Out For

- **`onChange` timing:** `onChange(of: selectedSectionId)` fires after `onAppear`. Make sure `onAppear` sets `selectedGroupId` directly rather than relying on the `onChange` to fire.
- **`initialStack` with `initialSection`:** When both are provided (e.g., from a stack context menu in Epic 8), both section and stack pickers should be hidden. Check: `if initialSection == nil` already hides the section picker. When `initialStack` is set, the stack picker should also be hidden. Add: `if sectionHasMultipleStacks && initialStack == nil`.
- **`ensureAllSectionsExist` still needed:** The save flow pushes all local sections/groups to the server before creating the protocol. This is still needed and should not be removed.

## Dev Notes

### Architecture Constraints
- Adding a protocol to a single-stack section must require zero extra taps vs current flow
- Stack picker only appears for multi-stack sections — no UX regression for simple cases
- `initialStack` parameter enables Epic 8's "Add Protocol Here" on stack context menus

### Patterns to Follow
- `.pickerStyle(.menu)` for both pickers (consistent with existing section picker)
- `.onChange(of:)` for cascading selection updates
