## User Story

**As a** dashboard user
**I want to** move protocols between stacks and stacks between sections, see per-stack progress, and feel haptic feedback when dragging
**So that** I can reorganize my routines fluidly with satisfying tactile feedback

Cross-moves use context menus with pickers (not drag-and-drop across sections, which is complex in SwiftUI). The batch reorder endpoint (Story 5.1) already accepts `section_id` on groups and `group_id` on protocols for cross-moves.

## Implementation Guidance

### Approach

Add context menus using SwiftUI `.contextMenu` on protocol rows and stack headers. Use a sheet or action sheet with a picker for target selection. Wire the selection to update the SwiftData model + call `syncPositions()`. Add a 20x20 circular progress ring to the stack header. Add `UIImpactFeedbackGenerator` calls in the existing reorder handlers.

### Layer Sequence

1. UI layer — modify views and shared components

### Step-by-Step

1. **Move protocol between stacks** — add context menu to each protocol row in `MasterTemplateEditor`:

```swift
NavigationLink { ... } label: { ProtocolRow(proto: proto) }
    .contextMenu {
        // Only show if section has multiple stacks
        if section.groups.count > 1 {
            Menu("Move to Stack…") {
                ForEach(section.sortedGroups.filter { $0.id != group.id }) { targetGroup in
                    Button(targetGroup.name) {
                        moveProtocol(proto, to: targetGroup)
                    }
                }
            }
        }
    }
```

Handler:
```swift
private func moveProtocol(_ proto: UserProtocol, to targetGroup: ProtocolGroup) {
    let maxPos = max((targetGroup.protocols.map(\.position).max() ?? -1), -1)
    proto.group = targetGroup
    proto.position = maxPos + 1
    SyncService.shared.syncPositions(protocols: [proto])
}
```

2. **Move stack between sections** — add to stack header context menu:

```swift
if sections.count > 1 {
    Menu("Move to Section…") {
        ForEach(sections.filter { $0.id != section.id }) { targetSection in
            Button(targetSection.name) {
                moveStack(group, to: targetSection)
            }
        }
    }
}
```

Handler:
```swift
private func moveStack(_ group: ProtocolGroup, to targetSection: ProtocolSection) {
    let maxPos = max((targetSection.groups.map(\.position).max() ?? -1), -1)
    group.section = targetSection
    group.position = maxPos + 1
    SyncService.shared.syncPositions(groups: [group])
}
```

### Gotchas / Watch Out For

- **Cross-move source stack may become empty:** After moving the last protocol out of a stack, the stack remains (empty). This is correct — the user can delete the empty stack via context menu if desired.
- **Cross-section stack move + single-stack rule:** Moving a stack into a section that had one stack means both stacks now show headers. Moving the last stack out of a section violates the "every section has at least one stack" rule. **Guard:** don't show "Move to Section…" if it's the last stack. Use `section.groups.count > 1` guard.
- **Haptic in onMove:** `onMove` fires when the drag completes (items have been reordered). The haptic fires at that moment — which is the "drop" moment. For a "pickup" haptic, you'd need a drag gesture modifier, which conflicts with `onMove`. Ship with drop-only haptic for now.
